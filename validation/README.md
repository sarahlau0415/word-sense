# Word Sense Validation Project

这个目录用于验证 Word Sense 在三个层面的真实能力:

- `使用`: 用户是否真的觉得它比查词典、问 ChatGPT、看小红书英语博主更有用。
- `增长`: 用户是否愿意主动分享、订阅、复访, 以及哪些入口最容易带来精准用户。
- `商业`: 哪些人有稳定需求, 为什么愿意付费, 愿意为什么形态付费。

当前判断: Word Sense 不是一个普通背单词工具, 更像一个面向中国成人英语学习者的跨文化语感产品。它的强项在于内容观点和解释方式, 风险在于使用频率、结果稳定性、规模化生产和付费场景是否足够明确。

## 项目结构

- `product-diagnosis.md`: 基于现有视觉页面和模型 prompt 的产品诊断。
- `hypotheses.md`: 需要被验证的核心假设。
- `experiment-backlog.md`: 按使用、增长、商业拆分的实验清单。
- `metrics.md`: 北极星指标、漏斗指标和质性信号。
- `user-research.md`: 访谈对象、问题和记录模板。
- `content-quality-rubric.md`: 词条内容质量评分表, 用来测试模型输出。
- `growth-validation.md`: 冷启动渠道、传播钩子和内容选题测试。
- `business-model.md`: 可测试的商业模式和定价假设。
- `test-plan.md`: 30 天验证路线图。

## 推荐验证顺序

1. 先验证一句话定位: “我为什么要用它, 而不是查字典或问通用 AI?”
2. 再验证内容质量: 同一个词条, 用户是否能说出他获得了什么新的理解。
3. 接着验证复访: 用户是否会在下一次遇到英文词时想起 Word Sense。
4. 最后验证付费: 稳定词汇知识需求来自哪里, 是个人、职业、学习还是内容创作。

## 当前最重要的 3 个问题

1. 用户会把 Word Sense 当作一次性的好奇阅读, 还是可重复使用的工具?
2. 用户愿意输入原句和出处吗? 如果不愿意, 产品能否用低摩擦方式帮他补足语境?
3. “深度讲一个词”能否形成自然分享, 还是需要改造成 newsletter、社群、课程、插件等更容易留存的形态?
