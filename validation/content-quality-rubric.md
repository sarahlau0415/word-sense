# Content Quality Rubric

每个词条按 1-5 分评分。低于 4 分的输出不建议进入公开内容池。

## 1. Sense 锚定

| 分数 | 标准 |
| --- | --- |
| 5 | 明确锚定用户原句里的 sense, 并说明其他场合可能不同 |
| 3 | 大体解释正确, 但有泛化倾向 |
| 1 | 把多个 sense 混在一起, 或解释错方向 |

## 2. 感觉是否成立

| 分数 | 标准 |
| --- | --- |
| 5 | 能让中文用户感到这个词的语气、姿态、使用场景 |
| 3 | 有解释, 但偏抽象 |
| 1 | 只是换一种方式写词典释义 |

## 3. 跟中文比

| 分数 | 标准 |
| --- | --- |
| 5 | 准确指出中文翻译漏掉、误带或缺失的层 |
| 3 | 有对比, 但不够锋利 |
| 1 | 只给中文翻译, 没有真正比较 |

## 4. 用法和例句

| 分数 | 标准 |
| --- | --- |
| 5 | 例句自然, 场域差异真实, 可以直接拿去用 |
| 3 | 例句正确但像教材 |
| 1 | 例句不自然、语域错误或过度表演网感 |

## 5. 来历和事实安全

| 分数 | 标准 |
| --- | --- |
| 5 | 词源/历史准确, 不编造具体人物事件 |
| 3 | 大体安全, 但信息太笼统 |
| 1 | 有可疑事实、虚构引用或过度确定 |

## 6. 悟道时刻

| 分数 | 标准 |
| --- | --- |
| 5 | 给出可迁移的识别模式, 用户能带走一把钥匙 |
| 3 | 有总结, 但像知识复盘 |
| 1 | 空泛鸡汤或没有必要的升华 |

## 7. 风格和节奏

| 分数 | 标准 |
| --- | --- |
| 5 | 像一个语感细腻的人在讲, 有判断但不说教 |
| 3 | 可读, 但 AI 感明显 |
| 1 | 机械、堆维度、过度标题化 |

## 推荐测试词池

### L1 低文化负载

`garment`, `bucket`, `refrigerator`, `aisle`, `drawer`

### L2 语感错位

`concerning`, `awkward`, `subtle`, `lowkey`, `vibe`, `cringe`, `mid`

### L3 概念负载

`agency`, `accountability`, `narrative`, `commitment`, `integrity`, `entitlement`

### 真实语境测试

每个词至少准备:

- 只有词
- 词 + 出处
- 词 + 出处 + 原句

这样才能测出模型是否真的会收窄 sense。
